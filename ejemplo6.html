07
Crear el Navbar con Link y localStorage

Dentro de src/components/ crea el archivo Navbar.jsx. El Navbar usa Link en lugar de <a> para navegar sin recargar la página. También lee el nombre guardado en localStorage para mostrarlo.

import { Link } from 'react-router-dom';

function Navbar() {
  const nombre = localStorage.getItem('nombreUsuario');

  return (
    <nav className="bg-indigo-700 text-white px-6 py-3 flex items-center gap-6">
      <span className="font-semibold text-base">MiApp</span>
      <div className="flex gap-4 flex-1">
        <Link to="/" className="text-white/80 hover:text-white text-sm">Inicio</Link>
        <Link to="/usuarios" className="text-white/80 hover:text-white text-sm">Usuarios</Link>
      </div>
      {nombre && (
        <span className="text-white/60 text-xs italic">Hola, {nombre}</span>
      )}
    </nav>
  );
}

export default Navbar;
Link de React Router funciona como un <a> pero intercepta el clic y actualiza la URL sin recargar la página — por eso la app se siente rápida. Usa to="" en lugar de href="".
localStorage.getItem('nombreUsuario') lee directamente del storage del navegador. Si el usuario aún no guardó su nombre, devuelve null y el saludo no aparece gracias al && condicional.
08
Configurar las rutas en App.jsx

Aquí se define qué componente renderiza React según la URL. BrowserRouter envuelve todo, Routes agrupa las rutas y cada Route define un par URL → componente.

import { BrowserRouter, Routes, Route } from 'react-router-dom';
import Home     from './pages/Home';
import Usuarios from './pages/Usuarios';
import Detalle  from './pages/Detalle';
import Navbar   from './components/Navbar';

function App() {
  return (
    <BrowserRouter>
      <Navbar />
      <div className="max-w-2xl mx-auto px-5 py-8">
        <Routes>
          <Route path="/"             element={<Home />}     />
          <Route path="/usuarios"     element={<Usuarios />} />
          <Route path="/usuarios/:id" element={<Detalle />}  />
        </Routes>
      </div>
    </BrowserRouter>
  );
}

export default App;
BrowserRouter debe envolver todo porque provee el contexto de navegación. Navbar está fuera de Routes para que siempre sea visible sin importar la ruta.
/usuarios/:id — los dos puntos antes de id indican que es un parámetro dinámico. Cualquier cosa que vaya ahí (/usuarios/1, /usuarios/42) renderizará el componente Detalle.
09
Desarrollar Home.jsx con localStorage

Home permite al usuario escribir su nombre y guardarlo en localStorage. Usa useEffect para cargar el nombre guardado al montar el componente.

import { useState, useEffect } from 'react';

function Home() {
  const [nombre, setNombre] = useState('');

  useEffect(() => {
    const guardado = localStorage.getItem('nombreUsuario');
    if (guardado) setNombre(guardado);
  }, []);

  function guardarNombre() {
    localStorage.setItem('nombreUsuario', nombre);
    alert(`Nombre guardado: ${nombre}`);
  }

  return (
    <div>
      <h2 className="text-2xl font-semibold mb-2">Bienvenido</h2>
      <p className="text-gray-500 text-sm mb-4">Escribe tu nombre para personalizar la app.</p>
      <div className="flex gap-3">
        <input
          type="text"
          placeholder="Tu nombre"
          value={nombre}
          onChange={(e) => setNombre(e.target.value)}
          className="flex-1 border border-gray-300 rounded-lg px-3 py-2 text-sm"
        />
        <button
          onClick={guardarNombre}
          className="bg-indigo-700 text-white px-5 py-2 rounded-lg text-sm"
        >
          Guardar
        </button>
      </div>
    </div>
  );
}

export default Home;
El useEffect con [] carga el nombre guardado al montar. Así si el usuario recarga la página, el input ya aparece con su nombre anterior.
⚠️ localStorage solo guarda strings. Si necesitas guardar un objeto o array, usa JSON.stringify() al guardar y JSON.parse() al leer.

Desarrollar Usuarios.jsx con Link

Muestra una lista de usuarios hardcodeados. Cada uno tiene un Link que navega a su página de detalle pasando el id en la URL.

import { Link } from 'react-router-dom';

const usuarios = [
  { id: 1, nombre: 'Ana García',    rol: 'Frontend Developer' },
  { id: 2, nombre: 'Luis Martínez', rol: 'Backend Developer'  },
  { id: 3, nombre: 'María López',   rol: 'UX Designer'        },
  { id: 4, nombre: 'Carlos Ruiz',   rol: 'DevOps Engineer'    },
];

function Usuarios() {
  return (
    <div>
      <h2 className="text-2xl font-semibold mb-2">Lista de usuarios</h2>
      <p className="text-gray-500 text-sm mb-4">Haz clic en un usuario para ver su perfil.</p>
      <ul className="flex flex-col gap-2">
        {usuarios.map((u) => (
          <li key={u.id} className="flex items-center justify-between border border-gray-200 rounded-lg px-4 py-3 bg-gray-50">
            <div>
              <p className="text-sm font-semibold">{u.nombre}</p>
              <p className="text-xs text-gray-400">{u.rol}</p>
            </div>
            <Link to={`/usuarios/${u.id}`} className="text-indigo-700 text-sm">
              Ver perfil →
            </Link>
          </li>
        ))}
      </ul>
    </div>
  );
}

export default Usuarios;
to={`/usuarios/${u.id}`} construye la URL dinámicamente con el id de cada usuario. Al hacer clic en Ana García, navega a /usuarios/1. Al hacer clic en Luis, navega a /usuarios/2.
11
Desarrollar Detalle.jsx con useParams

useParams lee el parámetro dinámico :id de la URL para saber qué usuario mostrar.

import { useParams, Link } from 'react-router-dom';

const usuarios = [
  { id: 1, nombre: 'Ana García',    rol: 'Frontend Developer', lenguaje: 'React'      },
  { id: 2, nombre: 'Luis Martínez', rol: 'Backend Developer',  lenguaje: 'Node.js'    },
  { id: 3, nombre: 'María López',   rol: 'UX Designer',        lenguaje: 'Figma'      },
  { id: 4, nombre: 'Carlos Ruiz',   rol: 'DevOps Engineer',    lenguaje: 'Kubernetes' },
];

function Detalle() {
  const { id } = useParams();

  const usuario = usuarios.find((u) => u.id === Number(id));

  if (!usuario) {
    return (
      <div>
        <p className="text-gray-500 mb-3">Usuario no encontrado.</p>
        <Link to="/usuarios" className="text-indigo-700 text-sm">← Volver a la lista</Link>
      </div>
    );
  }

  return (
    <div>
      <Link to="/usuarios" className="text-indigo-700 text-sm block mb-5">
        ← Volver a la lista
      </Link>
      <div className="border border-gray-200 rounded-xl p-6 bg-gray-50">
        <h2 className="text-xl font-semibold mb-1">{usuario.nombre}</h2>
        <p className="text-gray-500 text-sm mb-2">{usuario.rol}</p>
        <p className="text-sm text-gray-600">
          Herramienta favorita: <strong>{usuario.lenguaje}</strong>
        </p>
      </div>
    </div>
  );
}

export default Detalle;
useParams() devuelve un objeto con los parámetros de la URL. Como :id en la ruta, id llega como string — por eso usamos Number(id) para convertirlo antes de comparar con los ids del array que son números.
.find() recorre el array y devuelve el primer elemento que cumple la condición. Si ninguno cumple, devuelve undefined — por eso verificamos con el if (!usuario) antes de renderizar el perfil.

💡 ¿Te diste cuenta?
!
El nombre no se actualiza en el Navbar hasta que recargas la página

Si guardaste tu nombre en Home y luego miraste el Navbar, probablemente no cambió hasta que hiciste F5. Eso no es un bug — tiene una explicación lógica.

¿Qué está pasando?

El Navbar lee localStorage directamente cuando se monta por primera vez. Después de eso, nadie le avisa que el valor cambió — localStorage no es reactivo como useState. React solo re-renderiza un componente cuando cambia su estado o sus props. Como el Navbar no tiene estado propio ni recibió props nuevas, no tiene razón para volver a ejecutarse y leer el valor actualizado.

En otras palabras: localStorage guarda el dato, pero no le dice a nadie que cambió.

La solución es aplicar el patrón de lifting state up que ya conoces — subir el estado del nombre a App.jsx para que tanto Home como Navbar compartan la misma fuente de verdad.

El estado del nombre sube a App.jsx — que es el padre común de Navbar y Home. Home recibe una función onGuardar para notificar el cambio, y Navbar recibe el nombre como prop en lugar de leerlo del localStorage.
App.jsx — agrega el estado y pásalo a los componentes
import { useState } from 'react';
import { BrowserRouter, Routes, Route } from 'react-router-dom';
import Home     from './pages/Home';
import Usuarios from './pages/Usuarios';
import Detalle  from './pages/Detalle';
import Navbar   from './components/Navbar';

function App() {
  const [nombre, setNombre] = useState(
    localStorage.getItem('nombreUsuario') || ''
  );

  return (
    <BrowserRouter>
      <Navbar nombre={nombre} />
      <div className="max-w-2xl mx-auto px-5 py-8">
        <Routes>
          <Route path="/"             element={<Home onGuardar={setNombre} />} />
          <Route path="/usuarios"     element={<Usuarios />} />
          <Route path="/usuarios/:id" element={<Detalle />}  />
        </Routes>
      </div>
    </BrowserRouter>
  );
}

export default App;
Navbar.jsx — recibe nombre como prop, ya no lee localStorage
import { Link } from 'react-router-dom';

function Navbar({ nombre }) {
  return (
    <nav className="bg-indigo-700 text-white px-6 py-3 flex items-center gap-6">
      <span className="font-semibold text-base">MiApp</span>
      <div className="flex gap-4 flex-1">
        <Link to="/" className="text-white/80 hover:text-white text-sm">Inicio</Link>
        <Link to="/usuarios" className="text-white/80 hover:text-white text-sm">Usuarios</Link>
      </div>
      {nombre && (
        <span className="text-white/60 text-xs italic">Hola, {nombre}</span>
      )}
    </nav>
  );
}

export default Navbar;
Home.jsx — recibe onGuardar como prop y lo llama al guardar
import { useState } from 'react';

function Home({ onGuardar }) {
  const [nombre, setNombre] = useState(
    localStorage.getItem('nombreUsuario') || ''
  );

  function guardarNombre() {
    localStorage.setItem('nombreUsuario', nombre);
    onGuardar(nombre);  // notifica a App del cambio → App re-renderiza → Navbar se actualiza
  }

  return (
    <div>
      <h2 className="text-2xl font-semibold mb-2">Bienvenido</h2>
      <p className="text-gray-500 text-sm mb-4">Escribe tu nombre para personalizar la app.</p>
      <div className="flex gap-3">
        <input
          type="text"
          placeholder="Tu nombre"
          value={nombre}
          onChange={(e) => setNombre(e.target.value)}
          className="flex-1 border border-gray-300 rounded-lg px-3 py-2 text-sm"
        />
        <button
          onClick={guardarNombre}
          className="bg-indigo-700 text-white px-5 py-2 rounded-lg text-sm"
        >
          Guardar
        </button>
      </div>
    </div>
  );
}

export default Home;
Ahora cuando el usuario guarda su nombre: Home llama a onGuardar(nombre) → App actualiza su estado con setNombre → React re-renderiza App → Navbar recibe el nombre nuevo como prop → el saludo aparece inmediatamente sin recargar.

🎯 Reto extra
Página 404 para rutas desconocidas

Agrega una ruta comodín que capture cualquier URL que no coincida con las rutas definidas y muestre una página de error amigable con un botón para volver al inicio.

src/pages/NotFound.jsx
import { Link } from 'react-router-dom';

function NotFound() {
  return (
    <div className="pagina">
      <h2>404 — Página no encontrada</h2>
      <p>La ruta que buscas no existe.</p>
      <Link to="/">← Volver al inicio</Link>
    </div>
  );
}

export default NotFound;
App.jsx — agregar la ruta al final de Routes
import NotFound from './pages/NotFound';

// Dentro de <Routes>, al final:
<Route path="*" element={<NotFound />} />
path="*" es el comodín — captura cualquier URL que no haya coincidido con las rutas anteriores. React Router siempre evalúa las rutas en orden, así que el comodín solo se activa si ninguna otra ruta coincidió.
