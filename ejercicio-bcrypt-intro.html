Semana 6

Mini ejercicio: ¿para qué sirve bcryptjs?

Node.js bcryptjs hash 15–20 min
Objetivo
00
Ver en consola qué hace un hash y una comparación

En el registro de usuarios no debes guardar la contraseña en texto plano. bcrypt genera un hash: un string largo que no se puede “deshacer” para recuperar la contraseña original, pero sí permite comprobar si otra cadena (la que el usuario escribe al iniciar sesión) coincide con la que se usó al crear el hash.

Este mini ejercicio es aparte del starter de auth: solo necesitas una carpeta vacía y Node para ejecutar un script de unas líneas.

Pasos
01
Crear el proyecto mínimo e instalar bcryptjs

Crea una carpeta (por ejemplo demo-bcrypt), ábrela en la terminal y ejecuta:

Terminal
npm init -y
npm install bcryptjs
02
Crear el archivo del script

En esa misma carpeta, crea demo-bcrypt.js con el siguiente contenido:

demo-bcrypt.js
const bcrypt = require('bcryptjs');

async function main() {
  const password = 'secreta123';

  const hash1 = await bcrypt.hash(password, 10);
  const hash2 = await bcrypt.hash(password, 10);

  console.log('Misma contraseña, dos hashes:');
  console.log(hash1);
  console.log(hash2);
  console.log('¿Los dos strings son idénticos?', hash1 === hash2);

  console.log('Compare con la contraseña correcta:', await bcrypt.compare(password, hash1));
  console.log('Compare con otra contraseña:', await bcrypt.compare('otraClave', hash1));
}

main();
El número 10 son las salt rounds: más alto, más costoso de calcular y más lento. En muchos proyectos se usa entre 10 y 12.
03
Ejecutar y observar
Terminal
node demo-bcrypt.js

Comprueba en la salida que:

  • Los dos hashes son diferentes aunque la contraseña sea la misma (bcrypt incorpora salt aleatorio).
  • bcrypt.compare con la contraseña correcta devuelve true.
  • Con una contraseña incorrecta devuelve false.
Relación con el API de auth: en registro guardas en la base de datos el resultado de bcrypt.hash. En login usas bcrypt.compare con lo que escribió el usuario y el hash almacenado.
