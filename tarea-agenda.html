📒 Agenda de Contactos

React Vite TailwindCSS ⏱ Entrega en 1 semana Individual
📋 Descripción

Construye una aplicación de agenda de contactos en React donde el usuario pueda agregar y eliminar contactos desde la misma pantalla.

La app debe tener al menos dos componentes separados además de App.jsx: uno para el formulario y otro para mostrar cada contacto. El estado de todos los contactos debe vivir en App.jsx.

Puedes usar el diseño visual que quieras — lo importante es que funcione correctamente y que el código esté organizado en componentes.

✅ Requisitos funcionales
📝
Formulario para agregar contactos
Debe tener al menos dos campos: nombre y teléfono. Al enviar, el nuevo contacto aparece en la lista. El formulario se limpia automáticamente después de agregar.
useState onChange onSubmit e.preventDefault()
🃏
Lista de contactos con tarjetas
Cada contacto se muestra en su propia tarjeta usando un componente separado. Los datos llegan via props. Se renderizan usando .map() con un key único por tarjeta.
props map() key componente
🗑️
Botón para eliminar contactos
Cada tarjeta tiene un botón que elimina ese contacto de la lista. Usa .filter() para crear el nuevo array sin el contacto eliminado.
filter() onClick prop de función
🔢
Contador de contactos
Muestra en algún lugar de la pantalla cuántos contactos hay guardados. El número se actualiza automáticamente cuando se agrega o elimina un contacto.
.length re-render
⚠️ El formulario no debe agregar un contacto si el campo de nombre está vacío. Valida con trim() antes de agregar.
🎯 Reto extra (opcional)
Filtro por categoría
Agrega un tercer campo al formulario para que el usuario elija una categoría: Trabajo, Familia o Amigo. Muestra un badge de color diferente en cada tarjeta según la categoría. Agrega botones para filtrar la lista y mostrar solo los contactos de una categoría.
Pista: necesitarás un segundo estado en App.jsx para guardar el filtro activo, y usar .filter() antes del .map().

🧩 Componentes sugeridos

Para esta tarea necesitas crear mínimo 2 componentes además de App.jsx. Aquí te describimos qué debería hacer cada uno — tú decides cómo programarlos.

1️⃣
ContactoForm.jsx — el formulario
Contiene los inputs y el botón de agregar. Maneja su propio estado interno para los campos de texto. Cuando el usuario hace submit, llama a una función que le pasa el padre (App.jsx) para agregar el contacto al array.

Props que recibe: una función onAgregar que viene de App.jsx.
2️⃣
ContactoCard.jsx — la tarjeta de un contacto
Muestra la información de un solo contacto: nombre, teléfono, y un botón de eliminar. No tiene estado propio — solo recibe datos via props y llama a funciones del padre cuando el usuario interactúa.

Props que recibe: nombre, telefono, y una función onEliminar.
3️⃣
ContactoList.jsx — la lista (opcional)
Si quieres separar mejor las responsabilidades, puedes crear un tercer componente que reciba el array de contactos y se encargue de renderizar las tarjetas con .map(). Esto deja App.jsx más limpio.

Props que recibe: contactos (el array) y onEliminar (la función).
Una buena regla general: si un componente empieza a tener más de 50 líneas, probablemente se puede dividir en dos. Para esta tarea, con 2 o 3 componentes es más que suficiente.

  • Puedo agregar un contacto nuevo desde el formulario

  • Cada contacto tiene su propio botón de eliminar y funciona
  • El contador de contactos refleja el número correcto en todo momento
  • Hay al menos dos componentes además de App.jsx
  • El estado de los contactos vive en App.jsx
