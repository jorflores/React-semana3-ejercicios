Semana 6

Mini ejercicio: firmar y verificar un JWT con jsonwebtoken

Node.js jsonwebtoken JWT 15–20 min
Objetivo
00
Ver en consola cómo se crea un token y cómo se valida

Un JWT es un string con tres segmentos separados por punto: cabecera, payload (datos como id o email, codificados en base64) y firma. La firma depende de un secreto compartido: si cambias el payload o el secreto no coincide, jwt.verify falla.

Ejercicio aparte del starter de auth: carpeta nueva y Node. Sirve para entender lo que hará después tu ruta de login al emitir el token y tu middleware al verificarlo.

En el script usamos un secreto fijo solo para practicar. En el API real usa process.env.JWT_SECRET y nunca subas el secreto al repositorio.
Pasos
01
Proyecto mínimo e instalar jsonwebtoken

Crea una carpeta (por ejemplo demo-jwt), ábrela en la terminal y ejecuta:

Terminal
npm init -y
npm install jsonwebtoken
02
Crear el archivo del script

Crea demo-jwt.js con el siguiente contenido:

demo-jwt.js
const jwt = require('jsonwebtoken');

// Solo para este laboratorio local (en el API: process.env.JWT_SECRET)
const SECRET = 'demo-secret-laboratorio-jwt';

async function main() {
  const token = jwt.sign(
    { id: 42, email: 'alumno@ejemplo.com' },
    SECRET,
    { expiresIn: '2h' }
  );

  console.log('El token tiene tres partes:', token.split('.').length);
  console.log('Token completo:\n', token);
  console.log();

  console.log('jwt.decode (no comprueba la firma):', jwt.decode(token));
  console.log();

  const verificado = jwt.verify(token, SECRET);
  console.log('jwt.verify con el mismo SECRET:', verificado);
  console.log();

  try {
    jwt.verify(token, 'clave-completamente-distinta');
  } catch (e) {
    console.log('Secreto incorrecto:', e.name, '-', e.message);
  }

  const casiExpirado = jwt.sign({ demo: true }, SECRET, { expiresIn: '1s' });
  console.log('Esperando a que expire el token de 1s...');
  await new Promise((r) => setTimeout(r, 1200));
  try {
    jwt.verify(casiExpirado, SECRET);
  } catch (e) {
    console.log('Token expirado:', e.name, '-', e.message);
  }
}

main();
jwt.decode sirve para inspeccionar el payload; no valida la firma. Quien protege rutas debe usar siempre jwt.verify con el mismo secreto con el que se firmó.
03
Ejecutar y observar
Terminal
node demo-jwt.js

Comprueba en la salida que:

  • El string tiene tres bloques separados por .
  • decode muestra el payload legible (no implica que el token sea confiable).
  • verify con el secreto correcto devuelve el objeto del payload.
  • Con un secreto distinto, verify lanza error.
  • Tras un segundo, el token de vida corta ya no pasa verify (expiración).
En tu API: el login usa jwt.sign para generar el token que envías al cliente; el middleware usa jwt.verify con la misma clave que en el login.
