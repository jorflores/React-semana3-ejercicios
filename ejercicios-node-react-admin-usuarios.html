Ejercicio Node 3

Frontend React para consumir el API de usuarios

React Vite TailwindCSS React Router Axios API REST 75-90 min
Objetivo
00
Conectar el frontend con el API creado en el ejercicio anterior

En este ejercicio usarás el proyecto react-admin-usuarios.zip, que ya viene solucionado con React, Vite, TailwindCSS, React Router y Axios.

No es necesario modificar el frontend: ya viene listo para conectarse al API. En esta práctica solo ajustarás tu backend del ejercicio anterior para permitir que la aplicación React pueda consumir sus rutas desde el navegador.

El backend debe ser el proyecto del ejercicio anterior: Express, Sequelize, Neon y rutas CRUD sobre /usuarios.
Rutas que consumirá el frontend
GET
/usuarios
Mostrar la lista de usuarios
GET
/usuarios/:id
Mostrar el detalle de un usuario
POST
/usuarios
Dar de alta un usuario
PUT
/usuarios/:id
Modificar un usuario existente
DELETE
/usuarios/:id
Eliminar un usuario
Preparar el backend
01
Verificar que el backend esté corriendo

Abre una terminal en el proyecto del API con Express, Sequelize y Neon. Ejecuta el backend para confirmar que sigue funcionando antes de conectar el frontend.

Terminal del backend
npm run dev

Valida en el navegador o Thunder Client:

URL de prueba
http://localhost:3000/usuarios
Si responde [] o una lista de usuarios, el backend está listo. Si no responde, corrige primero el backend antes de continuar.
02
Habilitar CORS en Express

El frontend de Vite normalmente corre en http://localhost:5173 y el backend en http://localhost:3000. Como son orígenes distintos, el navegador puede bloquear las peticiones si el backend no permite CORS.

Instala cors en el proyecto backend.

Terminal del backend
npm install cors

Después ajusta app.js del backend. El cambio importante es importar cors y registrar app.use(cors(...)) antes de las rutas.

app.js del backend
import express from 'express';
import dotenv from 'dotenv';
import cors from 'cors';
import sequelize from './config/database.js';
import usuarioRoutes from './routes/usuarioRoutes.js';
import './models/Usuario.js';

dotenv.config();

const app = express();
const PORT = process.env.PORT || 3000;

app.use(cors({
  origin: 'http://localhost:5173'
}));

app.use(express.json());
app.use(usuarioRoutes);

app.get('/', (req, res) => {
  res.json({ mensaje: 'API REST de usuarios con Express, Sequelize y Neon' });
});

try {
  await sequelize.authenticate();
  console.log('Conexión a Neon establecida correctamente');

  await sequelize.sync({ alter: true });
  console.log('Modelos sincronizados con la base de datos');

  app.listen(PORT, () => {
    console.log(`Servidor escuchando en http://localhost:${PORT}`);
  });
} catch (error) {
  console.error('Error al conectar con la base de datos:', error);
}
El orden importa: app.use(cors(...)) debe ir antes de app.use(usuarioRoutes). Si se coloca después, las rutas responden sin los headers de CORS y el navegador bloquea la petición.
03
Reiniciar y validar el backend

Después de instalar cors y modificar app.js, reinicia el backend. Si usas nodemon, guardar el archivo debe reiniciarlo automáticamente.

Terminal del backend
npm run dev

Vuelve a probar el endpoint:

URL de prueba
http://localhost:3000/usuarios
Si el backend responde correctamente, ya está listo para recibir peticiones desde el frontend.
04
Configurar y ejecutar el frontend ya solucionado

Descarga y descomprime react-admin-usuarios.zip. El frontend ya viene solucionado. Solo crea el archivo .env para indicarle cuál es la URL del API.

react-admin-usuarios/.env
VITE_API_URL=http://localhost:3000

Después ejecuta el proyecto para validar que puede consumir el backend.

Terminal del frontend
cd react-admin-usuarios
npm install
npm run dev

Abre la URL que indique Vite, normalmente:

URL del frontend
http://localhost:5173
No es necesario modificar archivos del frontend. Ya incluye React Router, TailwindCSS, Axios y las llamadas al API.
Validación final
05
Probar el flujo completo

Con backend y frontend corriendo al mismo tiempo, prueba el flujo completo desde la aplicación React:

  • Ir a /usuarios y ver la lista.
  • Crear un usuario desde /usuarios/nuevo.
  • Entrar al detalle desde /usuarios/:id.
  • Editar los datos desde /usuarios/:id/editar.
  • Eliminar desde /usuarios/:id/eliminar.
Si aparece un error de CORS, revisa que el backend tenga app.use(cors({ origin: 'http://localhost:5173' })) antes de las rutas.
