🪪 Generador de Tarjeta de Presentación

React Vite CSS ⏱ Entrega en 1 semana Individual
📋 Descripción

Construye una aplicación en React donde el usuario llene un formulario con su nombre y lenguaje de programación favorito, y al hacer clic en un botón se genere una tarjeta de presentación con esos datos.

La app debe vivir en un solo componente App.jsx. No se requieren componentes adicionales, pero puedes crearlos si lo deseas. El formulario debe limpiarse después de generar la tarjeta y debe haber un botón para resetear.

Puedes usar el diseño visual que quieras — lo importante es que funcione correctamente y que el código esté bien organizado.

✅ Requisitos funcionales
📝
Formulario con dos campos
Un input para el nombre y otro para el lenguaje favorito. Cada campo debe tener su propio estado y actualizarse con cada tecla que presiona el usuario.
useState onChange input controlado
🪪
Tarjeta generada condicionalmente
Al hacer clic en el botón, aparece una tarjeta con los datos ingresados. Antes de hacer clic, la tarjeta no debe estar visible. Usa un tercer estado booleano (mostrar) para controlar esto.
useState booleano renderizado condicional onClick
🔄
Botón Limpiar
Un botón que reinicia los tres estados a sus valores iniciales: nombre a '', lenguaje a '' y mostrar a false. La tarjeta desaparece y los inputs quedan vacíos.
onClick múltiples setters
🚫
Validación básica
El botón de generar no debe funcionar si alguno de los dos campos está vacío. Puedes deshabilitarlo con disabled o validar al hacer clic con un if antes de cambiar el estado.
trim() disabled condicional
⚠️ Usa trim() para validar que el usuario no haya escrito solo espacios en blanco en los campos.
🎯 Reto extra (opcional)
Avatar con iniciales + campo de rol
Agrega un tercer campo al formulario para que el usuario escriba su rol profesional (ej. Frontend Developer). Muéstralo en la tarjeta como un badge. Además, genera automáticamente las iniciales del nombre para mostrarlas en un círculo de avatar — si el nombre es "Ana García", las iniciales son "AG".
Pista para las iniciales: nombre.split(' ').map(p => p[0]).join('').toUpperCase()

🧩 Estados que necesitas

Esta app maneja tres estados en App.jsx. Antes de escribir código, entiende para qué sirve cada uno:

1
nombre — string
Guarda el texto que el usuario escribe en el primer input. Inicia en ''. Se actualiza con cada tecla usando onChange.
2
lenguaje — string
Guarda el texto del segundo input. Igual que nombre — inicia en '' y se actualiza con onChange.
3
mostrar — boolean
Controla si la tarjeta es visible o no. Inicia en false. Cambia a true cuando el usuario hace clic en "Generar tarjeta" y vuelve a false al limpiar.
📤 Checklist de entrega

  • El formulario tiene dos inputs controlados con useState

  • La tarjeta muestra el nombre y el lenguaje ingresados
  • El botón Limpiar oculta la tarjeta y vacía los inputs
  • No se puede generar una tarjeta con campos vacíos
  • La app tiene estilos básicos aplicados con CSS
