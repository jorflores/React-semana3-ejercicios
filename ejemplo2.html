Responsabilidad de cada uno

App.jsx

Define el array de alumnos y lo recorre con .map(). Crea un <TarjetaAlumno> por cada objeto del array.

TarjetaAlumno.jsx

Recibe nombre, lenguaje y activo como props. Muestra los datos y usa un ternario para mostrar "Activo" o "Inactivo".

Concepto clave

El mismo componente TarjetaAlumno se reutiliza N veces — una por alumno. Cada instancia recibe datos distintos pero usa el mismo código.

📋 Pasos del ejercicio
06
Crear el archivo del componente

Dentro de src/components/ crea el archivo TarjetaAlumno.jsx.
Si usas VS Code con snippets (p. ej. ES7+), puedes escribir rfce + Tab para generar la estructura base; si no, crea una función TarjetaAlumno que devuelva un <div> vacío y ajusta el nombre del componente.

07
Definir las props del componente

El componente recibe tres datos. Desestructúralos directamente en los parámetros de la función.

function TarjetaAlumno({ nombre, lenguaje, activo }) {
  return (
    <div className="tarjeta">
      <h3>{nombre}</h3>
      <p>Lenguaje favorito: {lenguaje}</p>
      <p>Estado: {activo ? 'Activo' : 'Inactivo'}</p>
    </div>
  );
}
Las props son de solo lectura — el componente puede mostrarlas pero nunca modificarlas directamente.
El operador ternario activo ? 'Activo' : 'Inactivo' es la forma de hacer condicionales dentro de JSX.
08
Exportar el componente

Al final del archivo, exporta el componente como default.

export default TarjetaAlumno;

10
Configurar App.jsx

Importa el CSS, importa el componente y define el array de alumnos con sus datos.

import './App.css';
import TarjetaAlumno from './components/TarjetaAlumno';

function App() {

  const alumnos = [
    { id: 1, nombre: 'Ana García',    lenguaje: 'JavaScript', activo: true  },
    { id: 2, nombre: 'Luis Martínez', lenguaje: 'Python',     activo: false },
    { id: 3, nombre: 'María López',   lenguaje: 'React',      activo: true  },
    { id: 4, nombre: 'Carlos Ruiz',   lenguaje: 'TypeScript', activo: true  },
  ];

  return (
    <div className="app-contenedor">
      <h1>Lista de Alumnos</h1>
    </div>
  );
}

export default App;

Agregar el CSS de App

12
Usar .map() para renderizar las tarjetas

Actualiza el return de App para iterar el array y renderizar un componente por cada alumno.

return (
  <div className="app-contenedor">
    <h1>Lista de Alumnos</h1>
    {alumnos.map((alumno) => (
      <TarjetaAlumno
        key={alumno.id}
        nombre={alumno.nombre}
        lenguaje={alumno.lenguaje}
        activo={alumno.activo}
      />
    ))}
  </div>
);
⚠️ key debe ser único y estable. Usamos alumno.id en lugar de index porque si el array cambia de orden, React puede confundirse con el índice.
13
Refactorizar con spread operator

Una vez que funcione, simplifica el .map() usando spread para pasar todas las props de golpe.

{alumnos.map((alumno) => (
  <TarjetaAlumno key={alumno.id} {...alumno} />
))}
{...alumno} expande todas las propiedades del objeto como props (incluido id). El componente solo usa las que desestructura; el resto se ignora. Si agregas un campo nuevo al objeto y lo muestras en la tarjeta, no hace falta cambiar el .map() en App.
🎯 Reto extra
Campo edad

Agrega un campo edad al array y muéstralo en la tarjeta. Solo requiere dos cambios: agregar edad a cada objeto del array y mostrarlo en el JSX del componente.

Array en App.jsx — agregar edad a cada objeto:
{ id: 1, nombre: 'Ana García',    lenguaje: 'JavaScript', activo: true,  edad: 22 },
{ id: 2, nombre: 'Luis Martínez', lenguaje: 'Python',     activo: false, edad: 25 },
{ id: 3, nombre: 'María López',   lenguaje: 'React',      activo: true,  edad: 20 },
{ id: 4, nombre: 'Carlos Ruiz',   lenguaje: 'TypeScript', activo: true,  edad: 23 },
TarjetaAlumno.jsx — agregar la prop y mostrarla:
function TarjetaAlumno({ nombre, lenguaje, activo, edad }) {
  return (
    <div className="tarjeta">
      <h3>{nombre}</h3>
      <p>Lenguaje favorito: {lenguaje}</p>
      <p>Edad: {edad} años</p>
      <p>Estado: {activo ? 'Activo' : 'Inactivo'}</p>
    </div>
  );
}
Si usas {...alumno} en el .map(), la prop edad llega automáticamente sin modificar el JSX de App.
