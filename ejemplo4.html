🕐 Reloj en Tiempo Real

useEffect useState setInterval cleanup array de dependencias ⏱ 20–25 min
🧩 Cómo funciona useEffect

Orden en que React ejecuta las cosas al montar, actualizar y desmontar un componente:

1. Monta
El componente aparece en pantalla por primera vez
→
2. Renderiza
React ejecuta el JSX y pinta la UI
→
3. useEffect
React ejecuta el efecto después de pintar

En este ejemplo:

Monta El componente Reloj aparece → el estado inicial es new Date()
Effect React crea el setInterval → cada 1 segundo llama a setHora(new Date())
Re-render Cada vez que hora cambia, React vuelve a pintar el JSX con la nueva hora
Cleanup Si el componente se desmonta, React ejecuta clearInterval → el intervalo se detiene
Montado — primera vez en pantalla
Effect — efecto secundario
Re-render — estado cambia
Cleanup — al desmontar

El segundo argumento de useEffect controla cuándo se vuelve a ejecutar el efecto:

Sintaxis ¿Cuándo corre? Caso de uso típico
useEffect(() => {}, []) Solo al montar Iniciar un intervalo, hacer un fetch inicial, suscribirse a un evento
useEffect(() => {}, [valor]) Al montar y cuando valor cambia Buscar datos cuando cambia el input del usuario, reaccionar a un prop
useEffect(() => {}) En cada render Raro — casi siempre es un error. Causa loops si el efecto actualiza estado.
En este ejemplo usamos [] porque el intervalo solo se necesita crear una vez. Si usáramos [hora], React crearía un nuevo intervalo cada segundo — generando un memory leak.

07
Importar useState y useEffect

Ambos hooks vienen del mismo paquete. Se importan juntos en la primera línea.

import { useState, useEffect } from 'react';
A diferencia de React 17 donde había que escribir import React from 'react', en React 18 con Vite solo importas los hooks que necesitas.
08
Declarar el estado de la hora

El estado guarda un objeto Date — no un string. El valor inicial es la hora en que el componente se monta.

const [hora, setHora] = useState(new Date());
new Date() crea un objeto con la fecha y hora actuales. Usamos los métodos .toLocaleTimeString() y .toLocaleDateString() para formatearlo después en el JSX.
09
Escribir el useEffect

Este es el corazón del ejemplo. Analiza cada parte con cuidado antes de escribirla.

useEffect(() => {

  const intervalo = setInterval(() => {
    setHora(new Date());
  }, 1000);

  return () => {
    clearInterval(intervalo);
  };

}, []);
setInterval ejecuta la función cada 1000ms (1 segundo). Dentro llama a setHora con la hora actual — esto dispara un re-render.
return () => clearInterval(intervalo) es la función de cleanup. React la ejecuta cuando el componente se desmonta para detener el intervalo y evitar memory leaks.
⚠️ El [] al final es obligatorio aquí. Sin él, React crearía un nuevo intervalo en cada render — generando cientos de intervalos en paralelo.
10
Escribir el JSX

Muestra la hora con toLocaleTimeString() y la fecha con toLocaleDateString() formateada en español.

return (
  <div className="reloj-contenedor">
    <p className="reloj-label">Hora actual</p>
    <h1 className="reloj-tiempo">
      {hora.toLocaleTimeString()}
    </h1>
    <p className="reloj-fecha">
      {hora.toLocaleDateString('es-MX', {
        weekday: 'long',
        year: 'numeric',
        month: 'long',
        day: 'numeric',
      })}
    </p>
  </div>
);
toLocaleDateString('es-MX', {...}) formatea la fecha en español mexicano con el día de la semana, mes y año completos. El resultado se ve como: lunes, 19 de mayo de 2025.

🎯 Reto extra
Botón de pausa y reanudación

Agrega un botón que pause y reanude el reloj. Requiere un segundo estado corriendo y agregar esa variable al array de dependencias del efecto.

const [corriendo, setCorriendo] = useState(true);

useEffect(() => {
  if (!corriendo) return;   // si está pausado, no crear el intervalo

  const intervalo = setInterval(() => {
    setHora(new Date());
  }, 1000);

  return () => clearInterval(intervalo);

}, [corriendo]);   // el efecto se re-ejecuta cuando corriendo cambia
// En el JSX, debajo del reloj:
<button onClick={() => setCorriendo(prev => !prev)}>
  {corriendo ? 'Pausar' : 'Reanudar'}
</button>
Ahora el array de dependencias tiene [corriendo]. Cuando el usuario pausa, corriendo cambia a false → React ejecuta el cleanup (clearInterval) y vuelve a correr el efecto → el if (!corriendo) return sale sin crear un nuevo intervalo.
